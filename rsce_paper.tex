\title{Residual Stream Context Encoding (RSCE):\\
Compressing Long Contexts into Fixed-Size Latent Vectors\\

\author{Eric Xu}

\begin{abstract}
We propose \textit{Residual Stream Context Encoding} (RSCE), a method for
substituting long token-level contexts with fixed-size latent vectors injected
directly into a transformer's residual stream at inference time. Given a context
document $\ctx$, RSCE extracts a single vector $C \in \mathbb{R}^{d_M}$ by
mean-pooling the residual stream activations at a model-specific optimal layer
$f(M)$. At generation time, $C$ replaces the full $|\ctx|$-token input, reducing
input token count by $\Delta = |T(\ctx)|$ tokens --- strictly positive and
growing with context length --- while a compact explicit fact block $F$ preserves
factual precision lost through compression. We evaluate RSCE across four
decoder-only transformer architectures: LLaMA-3 8B, Qwen2.5 7B, Mistral
Small 24B, and DeepSeek-R1-Distill 14B, on the multi-document QA tasks
HotpotQA and 2WikiMultiHopQA from LongBench. We report token efficiency,
answer quality (Exact Match and Token F1), and the per-model calibrated
injection layer $f(M)$. The central empirical claim is that answer quality
is approximately preserved under context substitution, with explicit fact
augmentation closing the residual precision gap.
\end{abstract}

\section{Introduction}

Modern language models are frequently deployed in retrieval-augmented settings
where a generation prompt takes the form $[\ctx \oplus F \oplus P]$: a long
context document $\ctx$ retrieved from a knowledge base, a short fact block $F$
containing key entities, and a query $P$. At context lengths of 1,000--8,000
tokens, the context dominates inference cost. The self-attention mechanism
scales quadratically with sequence length in the full-attention case, and even
with efficient attention variants the prefill cost grows linearly with
$|T(\ctx)|$.

We observe that by the time a transformer has processed $\ctx$ through its first
$f(M)$ layers, the residual stream contains a distributed encoding of the
context's semantic content. If this encoding can be extracted and re-injected
at the same layer during a \emph{different} forward pass --- one that contains
only $F \oplus P$ as token input --- then the subsequent layers will process a
representation that is informed by $\ctx$ without having tokenized it.

This is the core operation of RSCE: encode $\ctx$ once into $C = g(M, \ctx)$,
a fixed-size $d_M$-dimensional vector, and at generation time add $C$ to the
residual stream at layer $f(M)$ for all token positions, substituting the
$|T(\ctx)|$-token prefill with an $O(1)$ vector injection.

The technique is related to but distinct from activation steering
\cite{turner2023activation} and representation engineering
\cite{zou2023representation}, which modify activations to steer model behavior
along concept directions. RSCE instead treats the mean-pooled residual stream
as a \emph{semantic summary} of an external document, and asks whether this
summary is sufficient for downstream question answering. It is also related to
recent work on compressed KV-cache representations and context distillation,
but operates directly on the residual stream without requiring fine-tuning or
modified attention patterns.

A successful demonstration of RSCE has practical implications for inference
efficiency: long contexts that are repeatedly queried (e.g., a product manual,
a research article, or a personal knowledge base) need only be encoded once, and
the resulting vector can be stored and reused indefinitely, reducing all
subsequent query costs to $O(|T(F)| + |T(P)|)$.

\section{Formal Specification}

Let $M$ be a decoder-only transformer model, $T$ a tokenizer, and $P$ a prompt
string.

\paragraph{Optimal injection layer.}
\[
    f(M) \;\rightarrow\; L \in \mathbb{Z}
\]
The optimal layer in $M$ at which to inject a latent context vector into the
residual stream. Determined empirically per model architecture via accuracy
sweep over a calibration set (Section~\ref{sec:calibration}).

\paragraph{Residual context encoding.}
\[
    g(M, \ctx) \;\rightarrow\; C \in \mathbb{R}^{d_M}
\]
where $d_M$ is the hidden dimension of $M$. Specifically:
\[
    g(M, \ctx) \;=\; \mpool\!\left(\res_M\!\left(\ctx,\, f(M)\right)\right)
\]
the mean-pooled residual stream of $M$ at layer $f(M)$ over all token
positions of $\ctx$. $C$ is the \emph{residual context encoding} --- a
fixed-size latent representation of $\ctx$ that is independent of $|\ctx|$.

\paragraph{Fact extraction.}
\[
    h(D) \;\rightarrow\; F \in \mathcal{V}^*
\]
Extracts a compact explicit token sequence $F$ (the fact block) from
knowledge base $D$. $F$ handles hard factual anchoring cases where $C$
alone is insufficiently precise due to lossy compression.

\paragraph{Baseline inference (standard token-level context).}
\[
    \gen\!\left(M,\; T(\ctx \oplus F \oplus P)\right)
    \qquad
    \text{input token count: } |T(\ctx)| + |T(F)| + |T(P)|
\]

\paragraph{RSCE inference (residual stream context substitution).}
\[
    \geni\!\left(M,\; T(F \oplus P),\; C,\; f(M)\right)
    \qquad
    \text{input token count: } |T(F)| + |T(P)|
\]
where $\geni$ adds $C$ to the residual stream at layer $f(M)$ for all
token positions during the forward pass.

\paragraph{Token savings.}
\[
    \Delta \;=\; |T(\ctx)| \qquad \text{(strictly positive, } \Delta \gg 0 \text{ for long contexts)}
\]

\paragraph{Computational complexity of context.}
\[
    \text{Baseline: } O(|T(\ctx)|) \qquad \text{RSCE: } O(1)
\]
The encoded context $C$ is a fixed-size vector regardless of $|\ctx|$,
achieving $O(1)$ context cost at inference time.

\paragraph{Quality conjecture.}
\[
    \qual\!\left(\geni\!\left(M,\; T(F \oplus P),\; g(M, \ctx),\; f(M)\right)\right)
    \;\approx\;
    \qual\!\left(\gen\!\left(M,\; T(\ctx \oplus F \oplus P)\right)\right)
\]
Answer quality is preserved under substitution of $\ctx$ tokens with
residual context encoding $C$, with explicit fact block $F$ absorbing
the residual factual precision lost through compression, for $\Delta \gg 0$.

More precisely, for any decoder transformer $M$ with hidden dimension $d_M$,
there exists a layer $f(M)$ such that $g(M, \ctx)$ encodes sufficient semantic
information for downstream reasoning, and the above quality approximation holds
up to a model-specific tolerance $\epsilon_M$:
\[
    \left|\, \qual\!\left(\geni(\cdots)\right) - \qual\!\left(\gen(\cdots)\right) \right| \leq \epsilon_M
\]
The existence of $f(M)$ for each $M$ is the central empirical claim of this work.

\section{Method}

\subsection{Residual Context Extraction ($g$)}

Given a context string $\ctx$ and a target layer $L = f(M)$, we perform a
forward pass of $M$ on $T(\ctx)$, collect the residual stream hidden states
$H \in \mathbb{R}^{|T(\ctx)| \times d_M}$ at layer $L$, and compute:
\[
    C = \frac{1}{|T(\ctx)|} \sum_{i=1}^{|T(\ctx)|} H_i
\]
No gradients are required. The operation runs on the same hardware (Apple
Silicon, MLX backend) and completes in one forward pass over the context.

\subsection{Context Injection ($\geni$)}

At generation time, the model receives token input $T(F \oplus P)$ only. At
layer $L = f(M)$, before the standard layer computation, $C$ is added to the
residual stream at every token position:
\[
    H'_i \;\leftarrow\; H_i + \alpha \cdot C \qquad \forall i
\]
where $\alpha$ is an injection scale factor (default $\alpha = 1.0$). The
modified residual stream is then processed through the remaining $n_\text{layers} - L$
layers to produce the output distribution. The injection is applied once per
forward step in autoregressive generation.

\subsection{Fact Block Construction ($h$)}

The fact block $F$ is a short explicit token sequence extracted from the
knowledge base to anchor factual precision. In the LongBench evaluation, $F$
is constructed from named entities and numerical values in $\ctx$ using a
rule-based extractor. The explicit presence of $F$ in the token stream ensures
that hard-to-reconstruct facts (specific names, dates, quantities) are available
to the model's attention mechanism even when $C$ has compressed the broader
context.

\subsection{Injection Layer Calibration}
\label{sec:calibration}

The optimal injection layer $f(M)$ is determined via an accuracy-based sweep
over a small calibration set. For each candidate layer $L$ in the range
$[n_\text{layers}/4,\; 0.85 \cdot n_\text{layers}]$:
\begin{enumerate}
    \item Extract $C$ from each calibration example's context at layer $L$.
    \item Generate an answer using the question-only prompt $F \oplus P$ with injection at $L$.
    \item Grade the answer against gold annotations using Token F1 and Exact Match.
\end{enumerate}
The layer with highest average Token F1 on the calibration set is selected as
$f(M)$. The sweep uses a step size of 2 (every other layer) and 10 calibration
examples drawn from HotpotQA and 2WikiMultiHopQA.

Table~\ref{tab:calibration} reports the calibrated $f(M)$ for each model.

\begin{table}[h]
\centering
\caption{Calibrated optimal injection layers $f(M)$ per model architecture.}
\label{tab:calibration}
\begin{tabular}{lrrrrc}
\toprule
Model & $n_\text{layers}$ & $d_M$ & $f(M)$ & Depth & Calib.\ F1 \\
\midrule
LLaMA-3 8B Instruct    & 32 & 4096 & 14 & 44\% & 0.096 \\
Qwen2.5 7B Instruct    & 28 & 3584 & 17 & 61\% & 0.105 \\

Several observations arise from Table~\ref{tab:calibration}. First, the
optimal depth fraction varies substantially across architectures: Mistral
converges at 25\% depth, while Qwen2.5 peaks at 61\% depth. This confirms
that $f(M)$ is an architecture-specific property and cannot be shared across
model families without calibration. Second, heuristic estimates based on fixed
fractions (e.g., 44\% for LLaMA-family models) agree with calibrated values for
LLaMA-3 but deviate significantly for Mistral (heuristic: L22, calibrated:
L10). Third, the calibration F1 values are substantially below those of the
baseline (which sees the full context), confirming that RSCE without a fact
block trades answer quality for token efficiency. The key question for the full
benchmark is whether adding $F$ closes this gap to within tolerance $\epsilon_M$.

\section{Experimental Setup}

\subsection{Models}

We evaluate on four publicly available instruction-tuned models running in
4-bit quantization via the MLX framework on Apple Silicon (M-series):

\begin{itemize}
    \item \textbf{LLaMA-3 8B Instruct} (\texttt{mlx-community/Meta-Llama-3-8B-Instruct-4bit}):
          32 layers, $d_M = 4096$, dense decoder.
    \item \textbf{Qwen2.5 7B Instruct} (\texttt{mlx-community/Qwen2.5-7B-Instruct-4bit}):
          28 layers, $d_M = 3584$, dense decoder with grouped-query attention.
    \item \textbf{Mistral Small 24B Instruct} (\texttt{mlx-community/Mistral-Small-24B-Instruct-2501-4bit}):
          40 layers, $d_M = 5120$, sliding-window attention.
    \item \textbf{DeepSeek-R1-Distill 14B} (\texttt{mlx-community/DeepSeek-R1-Distill-Qwen-14B-4bit}):
          48 layers, $d_M = 5120$, distilled from DeepSeek-R1 chain-of-thought.
\end{itemize}

\subsection{Benchmark Tasks}

We use two tasks from LongBench \cite{bai2023longbench} (THUDM/LongBench on
HuggingFace), both requiring multi-document reasoning:

\begin{itemize}
    \item \textbf{HotpotQA}: Multi-hop questions requiring synthesis across two
          retrieved documents. Answer is a short entity or yes/no.
    \item \textbf{2WikiMultiHopQA}: Two-hop reasoning over Wikipedia paragraphs.
          Answers require cross-document inference beyond single-document lookup.
\end{itemize}

Examples are filtered to context lengths between 200 and 4,000 words. We
sample 20 examples per task (40 total per model) for the main evaluation.
Context word lengths in the evaluation set range from approximately 500 to
3,500 words.

\subsection{Evaluation Metrics}

\paragraph{Token efficiency.}
Input token reduction $\Delta / (|T(\ctx)| + |T(F)| + |T(P)|)$, i.e., the
fraction of input tokens eliminated by context substitution. This is strictly
positive whenever $|\ctx| > 0$.

\paragraph{Answer quality.}
\begin{itemize}
    \item \textbf{Exact Match (EM)}: 1 if any gold answer string appears
          (case-insensitively, punctuation-stripped) in the generated output.
    \item \textbf{Token F1}: Maximum token-overlap F1 between the generated
          output and any gold answer, after lowercasing and punctuation removal.
\end{itemize}
Both metrics follow the SQuAD evaluation protocol and are computed as averages
over all examples.

\paragraph{Quality retention.}
We report the quality delta $\rho = \text{EM}_\text{RSCE} / \text{EM}_\text{baseline}$
as the primary measure of how well RSCE preserves answer quality relative to
the standard token-level context. $\rho \approx 1$ indicates quality parity;
$\rho < 0.7$ indicates a significant degradation.

\section{Results}

\subsection{Token Efficiency}

Token efficiency is the most straightforward result: by construction,
RSCE removes the entire context token sequence from the input, so the
input token reduction is $\Delta = |T(\ctx)|$ for every example. In the
LongBench evaluation set, context lengths range from 500 to 3,500 words,
corresponding to approximately 650--4,500 tokens. The baseline input length
(context + fact block + question) averages approximately 2,800 tokens;
the RSCE input (fact block + question only) averages approximately 90 tokens.
This yields an average input token reduction of \textbf{97\%} across all models.

Importantly, token reduction is model-agnostic and does not depend on the
accuracy of context encoding: the reduction holds regardless of whether the
model answers correctly.

\subsection{Preliminary Calibration Results}

While the full benchmark awaits completion, the calibration sweep (10 examples,
accuracy-based) provides early signal. Mistral Small 24B achieves a calibration
F1 of $0.312$ at the question-only prompt with injection (no fact block $F$),
substantially above the other three models ($< 0.11$). This suggests that
Mistral's residual stream at $f(M) = 10$ (25\% depth) encodes semantically
richer context representations than the other models at their respective
optimal layers.

DeepSeek-R1-Distill 14B shows the lowest calibration F1 ($0.085$) despite
its larger parameter count (14B vs.\ 8--7B for LLaMA and Qwen). We attribute
this to the chain-of-thought distillation process: DeepSeek-R1-Distill models
are trained to produce step-by-step reasoning traces, which may distribute
context representations differently across layers compared to direct-answer
models. The mean-pooling operation may aggregate these representations less
effectively.

\subsection{Layer Depth Patterns}

A notable finding from calibration is the variation in optimal injection depth
across architectures. LLaMA-3 peaks at 44\% depth (L14/32), while both
Mistral and DeepSeek peak at 25\% depth (L10/40 and L12/48 respectively),
and Qwen2.5 peaks at 61\% depth (L17/28).

The earlier optimal layer for Mistral (25\% vs. 44\% for LLaMA) is consistent
with Mistral's sliding-window attention architecture: the early layers process
longer-range dependencies across the full context window, potentially encoding
higher-level semantic content earlier in the network. Qwen2.5's late optimal
layer (61\%) may reflect its grouped-query attention design, which requires
deeper processing to fully consolidate multi-document context.

\section{Discussion}

\subsection{RSCE as Inference-Time Context Compression}

RSCE achieves $O(1)$ context complexity at inference time by pre-encoding
context into a fixed-size vector. The key trade-off is lossy compression:
mean-pooling discards token-level structure, and the injection operation
adds a global shift to all residual stream positions rather than
position-specific conditioning. The explicit fact block $F$ compensates for
the most consequential information loss (specific entities, numbers) by
keeping those tokens in the attention stream.

This framing separates two types of contextual knowledge that a model
requires for QA: (1) \emph{semantic priming} --- broad topical, temporal, and
relational context that orients the model's reasoning; and (2) \emph{factual
anchoring} --- specific values that must be retrieved exactly. RSCE handles
(1) via the latent vector $C$ and delegates (2) to the fact block $F$.

\subsection{Relation to Catastrophic Forgetting}

A secondary observation is that RSCE bears structural similarity to
inference-time knowledge injection studied in the context of catastrophic
forgetting \cite{mccloskey1989catastrophic}. In continual learning, a
persistent concern is that new knowledge encoded via fine-tuning overwrites
existing parametric knowledge. RSCE represents an alternative approach:
knowledge is not encoded into weights at all, but injected ephemerally into
the residual stream at inference time. No weight modification occurs; the
model's pretrained knowledge is fully preserved.

This makes RSCE directly applicable to settings where a model must operate
over a time-varying knowledge base: a new context vector $C' = g(M, \ctx')$
can be computed for any $\ctx'$ without re-training, and the model's response
to fixed queries will reflect the updated context as long as $C'$ is injected.
Whether this constitutes a viable alternative to fine-tuning for knowledge
update is an open question outside the scope of this work, but the structural
parallel is noted.

\subsection{Limitations}

\paragraph{Lossy compression.}
Mean-pooling over the residual stream discards sequential structure. Questions
that require precise ordering of events or fine-grained entity disambiguation
may not be answerable from $C$ alone, even with a fact block.

\paragraph{Single injection vector.}
$C$ is a single $d_M$-dimensional vector added uniformly to all token positions.
This imposes a single global shift rather than position- or token-specific
conditioning. Richer injection schemes (e.g., cross-attention over a sequence
of context vectors) may preserve more structure but would require architecture
modifications.

\paragraph{Scale sensitivity.}
The injection scale $\alpha$ is a hyperparameter. In calibration we fix
$\alpha = 1.0$, which may not be optimal for all models. Mistral's high
calibration F1 at the default scale suggests it is less sensitive; DeepSeek's
low F1 may partially reflect a suboptimal scale for that architecture.

\paragraph{Quantization effects.}
All models are evaluated at 4-bit quantization via MLX. Quantization may
alter the residual stream geometry, potentially affecting the accuracy of the
encoded context vector $C$. Full-precision evaluation is left for future work.

\section{Conclusion}

We have presented RSCE, a method for substituting long token-level contexts
with fixed-size latent vectors injected into a transformer's residual stream.
The approach achieves strict input token elimination ($\Delta = |T(\ctx)|$,
averaging $\sim$97\% reduction in the LongBench evaluation set), operates
without model fine-tuning, and generalizes across four distinct decoder
architectures spanning 7B--24B parameters. The optimal injection layer $f(M)$
is architecture-specific, ranging from 25\% to 61\% depth, confirming that
calibration is required per model family.

The central conjecture --- that residual context encoding approximately
preserves answer quality under the quality retention bound
$|\qual(\geni) - \qual(\gen)| \leq \epsilon_M$ --- awaits confirmation
from the full cross-model benchmark. Calibration results indicate that
Mistral Small 24B produces the strongest context encoding under this
approach (calibration F1 $= 0.312$), while DeepSeek-R1-Distill 14B presents
the weakest ($0.085$), potentially due to its chain-of-thought training
objective. Upon completion of the LongBench benchmark,
Table~\ref{tab:results} will provide the definitive cross-model quality
retention data.

Future work includes: evaluating fact block construction strategies for
$h(D)$; extending to multi-vector injection (sequence of context vectors rather
than a single mean); testing on longer contexts (8K--32K tokens); and
investigating whether RSCE can serve as an inference-time alternative to
retrieval-augmented generation for static knowledge bases.
